# Проверка OTA: как обновить приложение и увидеть изменение

Под экраном «Учёт» добавлена строка: **«✓ Обновления через OTA включены»**.  
Если после обновления через OTA вы видите эту надпись — обновление прошло успешно.

При установке обновления показывается экран с анимацией: сначала «Загрузка обновления...» с иконкой загрузки, затем «Готово! Перезапуск приложения.» — после этого приложение перезапускается.

---

## Что нужно перед тестом

1. На телефоне установлена **сборка с поддержкой OTA** (собранная с текущим `app.json`, где есть `expo-updates` и `updates.url`).
2. На компьютере в папке проекта выполнен **`npm install`** (чтобы работали команды EAS).

---

## Шаг 1: Опубликовать обновление (на компьютере)

В корне проекта откройте терминал и выполните **одну** команду — в зависимости от того, каким профилем вы собирали приложение.

**Если ставили приложение из production-сборки:**

```bash
eas update --branch production --message "Тест OTA: надпись под Учет"
```

**Если ставили APK (профиль apk):**

```bash
eas update --branch apk --message "Тест OTA: надпись под Учет"
```

Дождитесь сообщения об успешной публикации (обычно 1–2 минуты).

---

## Шаг 2: Получить обновление на телефоне

**Вариант А (удобный)**  
1. Откройте приложение.  
2. Перейдите: **Статистика** → иконка сохранения (резервная копия) вверху.  
3. На экране «Резервная копия» нажмите **«Проверить обновления»**.  
4. Если появится диалог «Найдено обновление» — нажмите **«Обновить»**.  
5. Приложение перезапустится.

**Вариант Б**  
1. Полностью закройте приложение (уберите из недавних).  
2. Снова откройте приложение.  
3. При запуске может появиться диалог об обновлении — подтвердите установку.

---

## Шаг 3: Проверить результат

Откройте вкладку **«Учёт»**.

Под блоком с датой и парой должна быть строка:

**«✓ Обновления через OTA включены»**

Если она есть — OTA работает, обновление применилось.

---

## Если обновление не появилось

- Убедитесь, что команду `eas update` выполнили с **тем же каналом**, каким собирали приложение (production или apk).
- Ещё раз нажмите «Проверить обновления» на экране Резервная копия или перезапустите приложение.
- Проверьте интернет на телефоне.
- Если приложение было установлено старым билдом (до настройки OTA), оно не получит обновления — нужна новая сборка (см. `docs/BUILD_AND_OTA_CHECKLIST.md`).

---

## Кратко

1. На ПК: `eas update --branch production --message "Тест OTA"` (или `apk` вместо `production`).  
2. На телефоне: Резервная копия → «Проверить обновления» → Обновить.  
3. Открыть «Учёт» и увидеть надпись «✓ Обновления через OTA включены».
