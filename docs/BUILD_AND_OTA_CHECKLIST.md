# Чеклист: новый билд с гарантированной поддержкой OTA

Чтобы установленное на телефоне приложение получало обновления без переустановки из магазина, билд **должен быть собран** с уже включённой конфигурацией EAS Update. Ниже — пошаговый чеклист.

---

## Что уже сделано в проекте (проверено)

- В **app.json**:
  - `runtimeVersion: { "policy": "appVersion" }` — версия из app.json (сейчас 1.0.3)
  - `updates.enabled: true`
  - `updates.url` указывает на ваш проект EAS
  - `updates.checkAutomatically: "ON_LOAD"` — проверка при каждом запуске
  - Плагин **expo-updates** в `plugins`
- В **eas.json** у каждого профиля указан **channel** (production, apk, preview)
- В приложении: **UpdateChecker** при запуске проверяет обновления; на экране «Резервная копия» есть кнопка **«Проверить обновления»** и отображается версия приложения

Если билд был собран **до** добавления этих настроек, старый APK/IPA не будет получать OTA. Нужен **новый билд** по инструкции ниже.

---

## Шаг 1: Установить зависимости

В корне проекта:

```bash
npm install
```

Без этого `eas build` и `eas update` могут падать с ошибками про конфиг.

---

## Шаг 2: Собрать новый билд

Канал обновлений задаётся профилем сборки. Какой профиль использовать — зависит от того, как вы ставите приложение.

### Вариант A: Сборка для магазинов / внутренняя production-сборка

```bash
eas build --profile production --platform android
```

Или для двух платформ:

```bash
eas build --profile production --platform all
```

В этом билде будет вшит **channel: production**. Все последующие OTA нужно публиковать в канал **production**.

### Вариант B: Отдельный APK (без магазина)

```bash
eas build --profile apk --platform android
```

В билде будет **channel: apk**. OTA публиковать в канал **apk**.

### Удобные скрипты из package.json

```bash
npm run build:production
# или
npm run build:apk
```

Дождитесь окончания сборки в EAS, скачайте и установите билд на устройство.

---

## Шаг 3: После установки билда — опубликовать первое OTA (опционально)

Чтобы сразу проверить обновления, можно опубликовать текущий код как обновление:

**Если собирали production:**

```bash
eas update --branch production --message "Версия 1.0.3: нумерация студентов, OTA, pull-to-refresh в журнале"
```

**Если собирали apk:**

```bash
eas update --branch apk --message "Версия 1.0.3: нумерация студентов, OTA, pull-to-refresh в журнале"
```

Или через скрипты (нужно дописать сообщение вручную):

```bash
npm run update:production -- "Ваше сообщение"
npm run update:apk -- "Ваше сообщение"
```

---

## Шаг 4: На телефоне получить обновление

1. Открыть приложение.
2. Зайти: **Статистика** → иконка сохранения (резервная копия) → экран **«Резервная копия»**.
3. Нажать **«Проверить обновления»**.
4. Если обновление есть — подтвердить установку; приложение перезапустится с новой версией.

Либо полностью закрыть приложение и снова открыть — проверка выполняется при каждом запуске.

---

## Соответствие канала и профиля сборки

| Профиль сборки   | Канал в eas.json | Команда публикации OTA                    |
|------------------|------------------|-------------------------------------------|
| production       | production       | `eas update --branch production --message "..."` |
| apk              | apk              | `eas update --branch apk --message "..."`       |
| preview          | preview          | `eas update --branch preview --message "..."`    |

Публиковать обновления нужно **в тот канал, которым собирали билд**.

---

## Частые проблемы

**На телефоне ничего не обновляется**

- Установленный билд был собран **до** добавления expo-updates и `updates` в app.json → нужен **новый билд** по шагам выше.
- OTA опубликован в другой канал (например, собрали apk, а обновление выложили в production) → публиковать в тот же канал, что и у билда.
- На устройстве не нажимали «Проверить обновления» и не перезапускали приложение после публикации OTA.

**Ошибка при `eas update` или `eas build`**

- Выполнить `npm install` в корне проекта.
- Проверить, что в app.json есть `expo-updates` в `plugins` и блок `updates` с `url` и `enabled: true`.

**В браузере изменения есть, на телефоне — нет**

- Браузер всегда грузит код с вашего ПК. Телефон использует только то, что в билде + OTA. Нужно: новый билд (если старый без OTA) и публикация OTA в правильный канал, затем на телефоне — «Проверить обновления» или перезапуск приложения.

---

## Краткая последовательность «всё с нуля»

1. `npm install`
2. `eas build --profile production --platform android` (или apk, если нужен только APK)
3. Установить полученный билд на телефон
4. `eas update --branch production --message "Описание"` (или apk)
5. В приложении: Резервная копия → «Проверить обновления»

После этого все последующие правки в JS/TS можно распространять только через `eas update`, без пересборки билда (пока не меняете нативный код, SDK или app.json в части нативных настроек).
