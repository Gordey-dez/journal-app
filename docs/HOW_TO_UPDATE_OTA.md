# Как обновить приложение через OTA

## Шаг 1: Внесите изменения в код

Вы уже внесли изменения в код (добавили нумерацию студентов). Теперь нужно опубликовать обновление.

## Шаг 2: Убедитесь, что вы в правильной директории

```bash
cd C:\Users\ABOBUS\attendance-app-clean
```

## Шаг 3: Установите EAS CLI (если еще не установлен)

```bash
npm install -g eas-cli
```

## Шаг 4: Войдите в Expo аккаунт

```bash
eas login
```

Или если уже вошли, проверьте:
```bash
eas whoami
```

## Шаг 5: Опубликуйте обновление

### Для Production (основной канал):

```bash
eas update --branch production --message "Добавлена нумерация студентов в разделе Учет"
```

### Для Preview (тестирование перед production):

```bash
eas update --branch preview --message "Добавлена нумерация студентов в разделе Учет"
```

## Шаг 6: Проверьте статус обновления

```bash
# Посмотреть список обновлений
eas update:list --branch production

# Или через веб-интерфейс
# Откройте https://expo.dev/accounts/gordy7/projects/attendance-app/updates
```

## Что происходит дальше?

1. **Обновление публикуется** на сервер Expo
2. **При следующем запуске** приложения пользователи увидят диалог об обновлении
3. **После подтверждения** приложение перезагрузится с новым кодом
4. **Нумерация появится** в списке студентов

## Важно!

- ✅ Это изменение (нумерация) - это **JavaScript код**, его можно обновить через OTA
- ✅ Не требуется пересборка приложения
- ✅ Пользователям не нужно обновлять через магазин
- ✅ Обновление работает только для production builds (не в Expo Go)

## Проверка обновления локально

Если хотите протестировать обновление локально:

```bash
# Создайте development build
eas build --profile development --platform android

# Или для iOS
eas build --profile development --platform ios

# Затем опубликуйте обновление на development канал
eas update --branch development --message "Тест нумерации"
```

## Откат обновления (если нужно)

Если что-то пошло не так и нужно откатить:

```bash
eas update:republish --branch production
```

Или через веб-интерфейс Expo Dashboard можно откатить конкретное обновление.

## Мониторинг обновлений

## Часто задаваемые вопросы

**Q: Сколько времени занимает публикация?**  
A: Обычно 1-2 минуты.

**Q: Когда пользователи получат обновление?**  
A: При следующем запуске приложения (если включена автоматическая проверка).

**Q: Можно ли принудительно обновить?**  
A: Да, можно добавить кнопку "Проверить обновления" в настройках, используя `useUpdateChecker()`.

**Q: Что если обновление сломает приложение?**  
A: Можно откатить через `eas update:republish` или через Dashboard.
