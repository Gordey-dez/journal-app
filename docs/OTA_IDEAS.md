# Идеи для следующих обновлений (всё через OTA)

Всё ниже — только JS/UI, без новых нативных зависимостей. Можно выкатывать через `eas update`.

---

## Уже сделано в текущем релизе

- Pull-to-refresh на вкладке **Статистика**
- Экран **«О приложении»** (версия, проверка обновлений, ссылка на резервную копию)
- **Хаптик**: лёгкий отклик при нажатии «Сохранить» и успешном сохранении в Учёте, при «Проверить обновления» и кнопках на экранах Резервная копия и О приложении
- В **Журнале** быстрый фильтр **«Сегодня»** (одним нажатием показать занятия за сегодня)
- На экране **Резервная копия**: напоминание «Рекомендуем сделать бэкап» или «Последний бэкап был N дн. назад», если бэкапа не было или прошло больше 7 дней

---

## Что можно добавить позже (OTA)

### UX
- **Поиск по имени** на вкладке Студенты
- **Сортировка** в Статистике (по имени, по %, по пропускам)
- **Копировать в буфер** JSON экспорта с экрана Статистика (для быстрой передачи)
- **Светлая тема** (переключатель в настройках, тема только в JS/стилях)

### Данные
- **Заметка к занятию** (опциональное текстовое поле у занятия)
- **Быстрый выбор «Вчера»** в Журнале рядом с «Сегодня»
- Фильтр **«За последние 7 дней»** в Журнале

### Доступность
- **accessibilityLabel** и **accessibilityHint** у основных кнопок и списков (для screen reader)

### Полировка
- **Skeleton** или анимированный индикатор вместо текста «Загрузка...»
- Более информативные **пустые состояния** (картинка/текст, что делать дальше)
- **Подтверждение выхода** при несохранённом учёте уже есть; можно добавить то же для других экранов при необходимости

### Напоминания (без пушей)
- На экране Резервная копия уже показывается напоминание о давности бэкапа; можно усилить текст или порог (например, 3 дня)

---

## Что потребует нового билда

- Новые npm-пакеты с нативным кодом
- Смена иконки приложения или splash screen
- Push-уведомления (expo-notifications)
- Изменение прав в app.json (камера, геолокация и т.д.)
- Обновление версии Expo SDK
